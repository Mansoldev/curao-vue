TIPOS DE APLICACIONES (Agnósticas a la tecnología)

Por división de código

Antiguamente solo habia un tipo de aplicación web

  • La aplicación monolito: donde estaba desde la conexión a BBDD a las vistas

Hoy en día solemos encontrar al menos 2

  • La aplicación Front: HTML, CSS y JS (Excepciones como Blazor)
  • La API: Hecha con cualquier lenguaje de Back y normalmente envia y recibe JSON (xml si es vieja)
Por utilidad

Los tipos de aplicaciones más comunes:

  • Web corporativa (Landing): cualquier web de empresa o particular con sus servicios
  • Webs de servicios a usuarios: Parecida a la anterior pero con más interacción (reserva de entradas o web de vuelos)
  • Enfocadas: Blogs, ecommerces/markets, portales educativos (Desarrollados bastantes con CMSs)
  • Aplicaciones especializadas: Teams, discord, vscode
  • Redes sociales
  • CRMs: Aplicaciones internas de empresas, cuentan con varios modulos
  • APIs públicas, privadas e integraciones

* Estos tipos de aplicaciones pueden ser cada una un monolito o estar separadas.

Por tecnologías - Front*
  • Estáticas: Sin backend (El resto dinámicas)
  • SPA: Single Page Aplication
  • SSR: Server Side Rendering
  • SSG: Static Site Generator
  • PWA: Progresive Web Apps
  • Híbridas: web y movil / Web y escritorio

Estos no son excluyentes

Por tecnologías - Back* (Arquitectura)

Normalmente depende de las capas en las que se componga

  • Código espagueti
  • MVC
  • TDD
  • DDD
  • Arquitectura hexagonal
  • Microservicios

TÉCNOLOGÍAS/BUENAS PRÁCTICAS DEL FRONT

Camino del front developper
HTML

Deberias de profundizar en los siguientes temas:

  • Atributos globales o no vistos
  • Etiquetas semánticas: header, nav, main, aside, footer, section, article, figure
  • Atributos Aria, Roles y accesibilidad
CSS

Deberias de profundizar en los siguientes temas:

  • Flexbox y Grid
  • Colores, fuentes, layout, selectores
  • Componentes avanzados, animaciones (Codepen)
  • Frameworks y librerias: Bootstrap, tailwind, normalize
  • Arquitectura CSS: BEM, ITCSS
  • Preprocesadores CSS (Sass) y PostCSS
JS - Lenguaje

Deberias de profundizar en los siguientes temas:

  • Eventos en elementos, de navegador, ratón, teclado, ventana, formulario
  • Funciones de manejo de objetos y arrays
  • Asincronia
  • Manipulacion del DOM
  • Llamadas a API: Fetch y axios
  • APIs de navegador
JS - Extendido

Deberias de profundizar en los siguientes temas:

  • JQuery, viejo pero lo pueden pedir
  • TypeScript
  • Manejarte con Node y NPM
  • Elegir un framework: Angular, React o Vue
  • Aprender ecosistema de framework: routeo, manejo de estado local y gloabl y de peticiones a API
  • Tests unitarios y end to end
Otras cosas de Front

Deberias de profundizar en los siguientes temas:

  • Diseño, prototipado (SIGMA) y wireframes
  • UX e UI
  • Performance
  • Ciclos del front
Hoy en día hay dos formas de manejar proyectos web de Front:
  • Creando un HTML e importando cosas con Links y Scripts
  • A través de NPM e importando paquetes

TÉCNOLOGÍAS/BUENAS PRÁCTICAS DEL BACK

Camino del Backend developper
Elegir un lenguaje

Deberias elegir un lenguaje y especializarte, pero probar varios en profundidad.

Buenas prácticas:
  • SOLID
  • Patrones de diseño
  • Composición de clases
  • Programación funcional
Saber como hacer varios tipos de aplicación
  • Un monolito
  • Microservicios
  • Una API
Para ello necesitas saber:
  • Autenticación y encriptación
  • Verbos HTTP
  • ORM
  • SQL
  • GIT
  • Sesiones, transacciones y persistencia
  • Test unitarios, funcionales, de integración
